# 避難所運営支援アプリ - CLAUDE.md

## プロジェクト概要

避難所に集まった住民数人が、QRコードを読むだけで役割分担し、1ステップずつ案内に従って避難所を立ち上げられるWebアプリ。

詳細は [docs/避難所運営支援アプリ_設計ドキュメント.md](docs/避難所運営支援アプリ_設計ドキュメント.md) を参照。

---

## このプロジェクトでの絶対ルール

### 設計思想(3原則)

1. **1画面に1ステップだけ**(認知負荷を最小に)
2. **「次にやることが指で押せる」**(読ませず、行動させる)
3. **判断を3択以下に**(できた / 困った / スキップ)

知識ゼロでも迷わないUIを最優先する。

### 技術スタック

- **フロント**: Next.js 14 (App Router) + TypeScript
- **スタイル**: Tailwind CSS のみ(外部UIライブラリは使わない)
- **バックエンド**: Supabase(認証・DB・リアルタイム同期)
- **配信**: PWA 対応(インストール不要、QRから即起動)
- **デプロイ先**: Vercel

### コーディング規約

- 関数コンポーネントのみ(クラスコンポーネント禁止)
- 状態管理は最初は `useState` / `useReducer`、必要になったら Zustand
- 日本語コメントOK(ただし **WHY** に限る、**WHAT** は書かない)
- ファイル名は kebab-case、コンポーネント名は PascalCase
- import alias は `@/*`(`src/` ディレクトリは使わない)
- モバイルファースト(まず 360px 幅で美しく)

### UI の数値ルール

- ボタンの最小高さ: **48px**(「できた」緑ボタンは **52px**)
- ステップ粒度: **1〜5分** で終わる単位(超える場合は分割)
- 手順表示: 最大 **3行**、各 **20文字以内**
- ステップタイトル: 動詞で終わる **10〜15文字**
- 高コントラスト、大きな文字、親指で押せるサイズ

## MVP スコープ(これ以外は実装しない)

### 対象範囲

- **役割**: 3つだけ(総務班 / 施設班 / 情報班)
- **災害**: 地震のみ
- **施設**: 学校体育館のみ
- **フェーズ**: 発災〜2時間(フェーズ 0〜2)
- **言語**: 日本語のみ
- **1セッション参加者**: 最大 10人
- **コンテンツ規模**: 3役割 × 8ステップ前後 = 約 25ステップ

### MVP に残す機能(7つだけ)

1. セッション生成(QR発行)
2. QR参加 + ニックネーム入力(ログインなし)
3. 役割推薦(3問質問 + 本人選択)
4. マイミッション画面(できた / 困った / スキップ)
5. 全体進捗ボード
6. 共有タイムライン(テキスト投稿のみ。写真は Supabase Storage の無料枠を消費するため MVP では見送り)
7. 訓練終了画面(完了ステップ・困った箇所のサマリー)

### やってはいけないこと

- 行政連携機能(MVP対象外、v2.0)
- 多言語対応(MVP対象外、v1.5)
- 複雑な認証(匿名 + ニックネームで十分)
- 食料物資 / 救護衛生 / 要配慮者役割の実装(MVP対象外、v1.0)
- 過度な機能追加(まず MVP 7機能の完成を優先)
- マニュアルの完全再現(行動に絞る)
- 個別の災害判断・医療判断をアプリが行うこと

## ディレクトリ構成

```
.
├── app/                  # Next.js App Router のページ
├── components/           # UI コンポーネント
├── lib/                  # ユーティリティ、Supabase クライアント、カスタムフック
│   ├── supabase/         # Supabase クライアント
│   └── hooks/            # カスタムフック
├── data/                 # ステップコンテンツ (JSON)
├── docs/                 # 設計ドキュメント
├── public/               # 静的ファイル(PWA manifest, アイコン等)
└── migrations/           # Supabase の SQL
```

## 開発フロー

- 1機能ずつブランチを切って実装
- コミットメッセージは日本語OK
- 大きな変更前にプランを共有
- **1画面ずつ作る**(一気に作らず分けて指示)
- 各画面ができたら `npm run dev` で実機(スマホ)確認

## ドメイン用語(設計ドキュメント付録より)

- **セッション**: 一つの避難所運営の単位。QR 1つに紐づく
- **役割**: 総務・施設・情報など、避難所運営上の班単位
- **ステップ**: 1〜5分で完了する具体的行動の最小単位
- **フェーズ**: 時間軸での運営段階。0(初動)〜4(撤収)
- **マイミッション**: 自分の役割で今やること。1画面1ステップ
